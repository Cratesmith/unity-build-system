Build Editor

Build steps

Create your own build steps. You can write pre and post build steps with custom filters and parameters. You build steps can even run async operations or trigger a recompile.

Scene selection

Each build process can have a custom selection of scenes, which are included in the build. This way you can easily include or exclude certain features.

Output path

Each build process needs an output path set. The build system stores these paths and automatically sets them for you.

Multi build

With the touch of a button you can trigger multiple build processes. Which will then automatically run one after the other.

Versioning

Your project includes a BuildVersion class. The build system can automatically update the build version for you. You can access the BuildVersion at runtime and display it in your game.

Pretend

To test your build steps, you can use the pretend mode, which does not actually call the build pipeline. If one of your steps fail the build system can be stopped by your code with a single call.

Build from commandline

If you use an automated build system like Jenkins you can run the build system via a commandline argument in Unity headless mode.

Free and Open Source

The build system is completely free and open source. And it will stay this way.

Works in Unity free

If you use Unity free, you can still use the build system. It will gently pause once you have to push the build button yourself and continue onwards automatically.

This is an open source project, so please feel free to contribute! Check out the source code to get started. The source is currently hosted at bitbucket.org
